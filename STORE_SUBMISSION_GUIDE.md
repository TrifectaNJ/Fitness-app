# App Store Submission Guide

## Google Play Store Submission

### 1. Prepare Your App
- Build signed AAB file using Android Studio
- Test on multiple Android devices
- Ensure app meets Google Play policies

### 2. Google Play Console Setup
1. Create developer account ($25 one-time fee)
2. Create new app in console
3. Fill out store listing:
   - App name: "Fitness Coach App"
   - Short description (80 chars)
   - Full description (4000 chars)
   - Screenshots (phone, tablet)
   - Feature graphic (1024x500)
   - App icon (512x512)

### 3. App Content
- Content rating questionnaire
- Target audience (age groups)
- Privacy policy URL (required)
- Permissions explanation

### 4. Release Management
- Upload AAB file
- Set version code and name
- Release notes
- Choose release track (internal/alpha/beta/production)

## Apple App Store Submission

### 1. Prepare Your App
- Archive app in Xcode
- Test on iOS devices
- Ensure App Store Review Guidelines compliance

### 2. App Store Connect Setup
1. Apple Developer account ($99/year)
2. Create new app
3. App information:
   - Bundle ID: com.fitnessapp.mobile
   - App name: "Fitness Coach App"
   - Subtitle (30 chars)
   - Description (4000 chars)
   - Keywords (100 chars)
   - Screenshots (all device sizes)
   - App icon (1024x1024)

### 3. App Review Information
- Contact information
- Demo account (if login required)
- Notes for reviewer
- Age rating

### 4. Release Management
- Upload IPA file via Xcode
- Set version and build number
- Release notes
- Choose release method (manual/automatic)

## Required Assets

### App Icons
- Android: 48x48, 72x72, 96x96, 144x144, 192x192, 512x512
- iOS: 20x20, 29x29, 40x40, 58x58, 60x60, 76x76, 80x80, 87x87, 120x120, 152x152, 167x167, 180x180, 1024x1024

### Screenshots
- Android: Phone (16:9), Tablet (16:10)
- iOS: iPhone (6.5", 5.5"), iPad (12.9", 11")

### Marketing Materials
- Feature graphic (Google Play)
- App preview videos (optional)
- Promotional text

## Legal Requirements

### Privacy Policy
Must include:
- Data collection practices
- Third-party services (Supabase)
- User rights
- Contact information

### Terms of Service
- App usage terms
- Account creation
- Content policies
- Liability limitations

## Timeline Expectations

### Google Play Store
- Review time: 1-3 days
- Policy violations: Additional review time
- Updates: Same day to 3 days

### Apple App Store
- Review time: 24-48 hours
- Rejections: 1-7 days for resubmission
- Updates: Same timeline as new apps

## Common Rejection Reasons

### Google Play
- Missing privacy policy
- Inappropriate content rating
- Broken functionality
- Policy violations

### Apple App Store
- Design guidelines violations
- Broken links/features
- Missing metadata
- Inappropriate content

## Post-Launch
